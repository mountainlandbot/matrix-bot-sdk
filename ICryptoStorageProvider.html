# getEventForMessageIndex

Gets the event ID for a previously successful decryption from a session and message index. If no event ID is known, this will return falsy. The caller can use this function to determine if a replay attack is being performed by checking the returned event ID, if present, against the event ID of the event it is decrypting. If the event IDs do not match but are truthy then the session may have been inappropriately re-used.
